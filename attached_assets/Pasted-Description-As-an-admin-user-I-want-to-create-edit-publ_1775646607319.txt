Description
As an admin user,
I want to create, edit, publish, and manage news, photos, videos, and categories,
so that website content can be maintained and displayed dynamically on the frontend.

Scope
Build the admin management interface for News & Gallery content.
This work must be implemented in the admin repository and connected to backend APIs and data models.
The admin must manage all content required for the current phase frontend.

Functional Requirements
1. Admin Sections
Add admin management sections for:

News
Photos
Videos
Categories
2. News Management
Admin must be able to:

create news item
edit news item
view news item
publish or save as draft
delete or archive news item if supported
assign category
upload/select cover image
enter publish date
enter short description
enter full content
3. Photos Management
Admin must be able to:

create photo item
edit photo item
view photo item
publish or save as draft
delete or archive photo item if supported
assign category
upload/select image
enter title
enter caption or description
enter publish date
Videos Management
Admin must be able to:
create video item
edit video item
view video item
publish or save as draft
delete or archive video item if supported
assign category
enter YouTube URL
enter title
enter short description
enter publish date
5. Categories Management
Admin must be able to:

create category
edit category
activate/deactivate category
use category in news, photos, and videos
6. Listing Pages in Admin
Each content type should have an admin list page with useful columns.
News list should show:

cover image thumbnail
title
category
publish date
status
last updated date
Photos list should show:
image thumbnail
title
category
publish date
status
Videos list should show:
title
YouTube URL or preview reference
category
publish date
status
7. Filters and Search in Admin
For current phase, admin should be able to:

search by title
filter by category
filter by status
8. Validation
Admin forms should validate required fields before save or publish.
Examples:

title is required
category is required where applicable
cover image is required for news
image is required for photo
YouTube URL is required for video
publish date required if publishing
9. Status Logic
For current phase:

draft content should not appear on public website
published content should appear in public APIs
10. Editor Support
News full content should support rich text editing.

11. Media Handling

Admin should upload/select image URLs or media references according to backend storage strategy
Do not store raw binary media directly in admin records if the backend uses file storage
Replit Implementation Notes

Build admin as production-ready management UI
Forms should be reusable where practical
Keep form and data structure ready for future enhancements
Do not fully implement future-phase features now
Keep UX clear and simple for content editors
Acceptance Criteria

Admin can manage news, photos, videos, and categories
Admin can create, edit, and publish content
Draft vs published behavior works correctly
Content saved in admin is reflected correctly through backend APIs
Admin lists and forms are usable and validated
News content supports full rich-text content entry
Future-Ready Notes
Admin structure should be ready for future support of:

featured news flag
multiple images per news article
tags
scheduled publish date/time
Arabic and English fields
SEO fields
manual related news selection
